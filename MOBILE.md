# Mobile apps (Android + iOS)

The same React app is packaged for phones with **Capacitor**. On mobile there is
no proxy, so the app talks to the router **directly over native HTTP**
(`src/api/transport.ts` uses `CapacitorHttp`, which bypasses the browser CORS
sandbox, sets the `Referer`/`Origin` the firmware requires, and keeps the login
cookie). Set your router IP in **Settings** if it isn't `192.168.0.1`.

> Your phone must be connected to the **router's Wi‑Fi** to reach it.

---

## Android (.apk)

### Option A — Cloud build (recommended, no tools to install)

A GitHub Actions workflow builds a downloadable APK for you:

1. Push this repo to GitHub (a private repo is fine).
2. Open the **Actions** tab → **Build Android APK** → **Run workflow**
   (it also runs automatically on every push to `main`).
3. When it finishes, open the run and download the **`ZTE-Router-Manager-apk`**
   artifact → unzip → `app-debug.apk`.
4. Copy it to your phone and install (enable *Install unknown apps* for your
   file manager/browser).

The workflow is `.github/workflows/android-apk.yml`.

### Option B — Build locally (needs Android Studio / SDK + JDK 17)

```bash
npm install
npm run build
npx cap add android          # first time only
npx cap sync android
# then either open Android Studio…
npx cap open android
# …or build from the CLI:
cd android && ./gradlew assembleDebug      # Windows: gradlew.bat assembleDebug
# → android/app/build/outputs/apk/debug/app-debug.apk
```

`npm run apk:debug` chains the build + Gradle steps on Windows.

## iOS (.ipa)

**iOS apps can only be built on macOS** — this is an Apple restriction, not a
project limitation. There is no way to produce an `.ipa` on Windows/Linux.

On a Mac:

```bash
npm install
npm run build
npx cap add ios              # first time only
npx cap sync ios
npx cap open ios             # opens Xcode
```

Then in Xcode: select your device, set a signing Team (a free Apple ID works for
7‑day sideloading; a paid Apple Developer account, US$99/yr, for longer), and
Run. For a distributable build use *Product → Archive*.

If you don't have a Mac, use a cloud macOS build service (e.g. Codemagic, Ionic
Appflow, or GitHub Actions `macos` runners) with the same steps.

## Notes

- The native folders `android/` and `ios/` are git‑ignored and regenerated by
  `npx cap add …` (and by the CI workflow), so the repo stays clean.
- `cleartext: true` in `capacitor.config.ts` is required because the router is
  plain `http://`.
- Everything stays on‑device: the phone talks only to your router over the LAN.
